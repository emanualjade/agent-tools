APPROACH: Lean-Adaptive Strike: Risk-Routed 6-Step Pipeline with a First-Class Architectural-Obstruction Loop
PHILOSOPHY: Fewest gates that still guarantee quality, with depth bought only where risk is proven. The current Strike pays an identical 5-step per-slice toll (research -> plan -> verify-plan -> build -> verify-build) for a CSS tweak and a payments ledger; that uniform ceremony is the central waste and, paradoxically, the central quality leak (a checklist run on everything trains the agent to skim it, so the real money path gets the same cursory pass as the copy fix). Lean-Adaptive replaces uniform depth with a single risk tier computed once at slice birth (TRIVIAL / STANDARD / CRITICAL) and a fast path vs full path keyed off it. Two principles are non-negotiable and survive verbatim: (1) the orchestration loop, never the agent, owns every stop/reassess/escalate trigger via mechanical counters; the agent never votes on whether it is stuck, done, or off-track. (2) Every spec/plan/slice is done only when a fresh context window can act on it without re-reading the transcript, and every gate emits a machine-readable PASS/FAIL/DONE with the failing criterion NAMED. On top of that the design front-loads the two things the original lacked: a READ-BEFORE-WRITE codebase-grounding pass (so the agent knows what already exists before it plans, killing duplication and missed validation layers at the source rather than catching them late), and a self-correcting ARCHITECTURAL-OBSTRUCTION loop that gives the build a reverse gear when an UPSTREAM decision (spec shape, phase boundary, shared contract) is wrong — the single biggest correctness gap, which the current engine literally logs as 'escalated route-back not auto-handled in v1'.
--- STEP SET (7) ---
  • 1. Spec + Phase Plan (merged, fresh-context gated) [Always runs (the only fully unconditional step). Phase-spec detail is folded in here for small phases; a separate phase-spec sub-pass is emitted inside Step 2 ONLY for phases flagged risk=critical or L+ size.] :: Produce ONE durable artifact that a fresh context window can build from: the main spec (outcome, the four plain boundaries In/Out-with-why/Needs-Decision/Never, domain language, product rules, state/lifecycle/invariants, engineering pressure points, binary acceptance-criteria checklist, success checks split repo-verifiable vs live/human) AND the ordered set of vertical phases — in a single pass for small initiatives, split only when the spec reveals genuine multi-phase scope.  [gate: PASS iff specPath written, acceptanceCriteria is non-empty and every item is binary/test-decidable, every phase is vertical (outcome-named, not a single layer), and the fresh-context self-check is asserted true. FAIL names the missing/ambiguous criterion or the horizontal phase.]
  • 2. Slice + Risk-Route (per phase) [Always runs per phase. The risk/path assignment here is the lever the whole pipeline pivots on.] :: Cut the phase into thin AND complete vertical slices, assign each a RISK TIER and SIZE that the rest of the pipeline consumes, and decide each slice's path (fast vs full). The first slice of the first phase must be a tracer bullet piercing every layer for the thinnest happy path with everything else stubbed.  [gate: PASS iff every slice names exactly one observable behavior with no 'and', leaves the app runnable, has a risk tier + path assigned, and the first slice of phase-01 is a tracer bullet. FAIL names the horizontal/groundwork/over-broad slice or the missing tracer bullet. If phaseSpecProblem is set, route back to Step 1 (upstream).]
  • 3. Plan-Build (merged for fast path; plan+canonical-research inline) + Altitude checkpoint [Always runs per slice. For TRIVIAL/STANDARD (fast path) this is research+plan+build in ONE agent round-trip (saves two round-trips vs the old 5-step). For CRITICAL (full path) build is split out and the step ends at a verified plan.] :: For a slice, produce a plan grounded in the EXISTING codebase, fold in canonical evidence only if a trigger surface fired, and (full path) name 2-3 candidate approaches before committing. This step MERGES research+plan and, for fast-path slices, continues straight into build.  [gate: PASS iff plan covers boundary+approach+test-plan, every new table passed the adjective-noun lens, canonical evidence is present for every detected trigger surface (or 'skipped' justified), and existing-code reuse was checked. If obstruction.detected, route per the obstruction loop instead of passing. FAIL names the unsearched duplication, the unjustified new table, or the missing canonical evidence on a money/auth surface.]
  • 4. Verify-Plan (CONDITIONAL — full path only) [CONDITIONAL: runs ONLY for full-path (CRITICAL) slices. CUT entirely for TRIVIAL/STANDARD slices — their plan defects are caught post-build by Step 6 (which already re-reads plan + criteria), removing one agent run + its fix loop from the majority of slices.] :: Independently verify a CRITICAL slice's plan before any code is written: canonical correctness, adjective-noun, blast radius, that the approach actually satisfies the named acceptance criteria.  [gate: PASS iff ready=true and every acceptance criterion the plan claims to cover is marked satisfiable and canonical/adjective-noun lenses pass. FAIL names the failing criterion or lens; routeBack targets plan (same slice) or upstream step.]
  • 5. Build (CONDITIONAL — full path only; fast path already built in Step 3) [CONDITIONAL: runs as a SEPARATE step only for full-path (CRITICAL) slices. For fast-path slices the build already happened inline in Step 3; this step is skipped.] :: Implement a CRITICAL slice's verified plan with one-hat discipline and surface-triggered guardrails, keeping every change traceable to a named acceptance criterion.  [gate: PASS iff built=true, every change maps to a named acceptance criterion (off-objective work is a stall signal, not just inefficiency), the two-hats invariant holds (no structure change in this commit), and every detected surface has its guardrail applied. FAIL or route back otherwise.]
  • 6. Verify-Build (honest ladder, risk-scaled) [Always runs per slice — the one universal gate. Fix loop re-runs THIS verifier (bounded by maxFixAttempts); for a single in-place fix the repair is appended to build.md rather than spawning a numbered fix file + fresh agent.] :: Verify the slice does its ONE observable behavior through the real entry point with real data, scaled to risk tier, and that it composes with prior slices in the phase. The single mandatory gate on every slice.  [gate: PASS iff verified=true, every acceptance criterion is PASS, the required ladder rung for the risk tier was actually reached (TRIVIAL: R1; STANDARD: R2/R3; CRITICAL: R2/R3 + negative probe + mandatory lenses), no test tampering found, and it composes with prior slices. FAIL names the failing criterion; fixNeeded routes to the fix loop (same verifier), routeBack escalates upstream if the plan/phase/spec is wrong.]
  • 7. Verify-Phase / Verify-Initiative (MERGED for single-phase) [For a SINGLE-PHASE initiative, verify-phase and verify-initiative COLLAPSE into one pass (run the cross-phase/readiness lenses as a delta), saving an entire final verify+fix round. For multi-phase, verify-phase runs per phase and verify-initiative runs once at the end.] :: Confirm the phase's slices integrate into the phase outcome, then (or simultaneously, for a single-phase initiative) confirm the whole initiative satisfies the main spec's acceptance criteria with one final behavior/screenshot pass.  [gate: PASS iff every main-spec acceptance criterion is PASS, cross-slice/cross-phase integration is observed working, and rollout safety is asserted (no broken half-feature live without a flag). FAIL names the failing criterion; an upstream routeBack re-enters Step 1/2 within budget.]
--- OBSTRUCTION LOOP ---
  detection: Detected at Step 3 (plan) or Step 5 (build) by SPECIFIC signals the agent must report as a structured field, NOT self-assessed prose: (a) the verified plan cannot be implemented without changing a file OUTSIDE the slice's named footprint; (b) the change requires touching a SHARED foundation (shared schema, base class, cross-module contract, public API consumed elsewhere); (c) a needed behavior change forces a structural change in the same edit (two-hats violation that cannot be cleanly separated within the footprint); (d) the same acceptance criterion cannot be satisfied because an UPSTREAM artifact (spec shape, phase boundary, a contract another phase assumes) is wrong or missing. PLUS loop-owned mechanical triggers the agent cannot rationalize away: the harness fingerprints (tool + target-region + hash-of-resulting-error); on 2 same-target edits OR 3 turns with no acceptance criterion moving FAIL->PASS OR 2 failed self-corrections, it fires a HARD INTERRUPT that injects this obstruction-routing step. Because a capable agent can perturb the target to dodge a pure fingerprint, the trigger ALSO fires on any edit that reaches outside the slice footprint or modifies a file flagged shared — a footprint/shared-surface tripwire that is harder to game than an error-hash window. The agent never decides whether reassessment happens; it only fills in the blast-radius/reversibility fields.
  decisionTree: Route STRICTLY by BLAST-RADIUS x REVERSIBILITY (no architectural taste required): TIER 1 — change is IN the slice's existing file footprint AND behavior-preserving -> do a preparatory refactor as a SEPARATE refactor commit (tests green before and after, zero behavior change), then resume the build. Bounded to files the slice already touches. TIER 2 — change needs a NEW seam but is REVERSIBLE (a two-way door: a facade/adapter/interface you can later replace without data loss or breaking consumers) -> insert an ENABLING SLICE behind the facade (thin, demoable, leaves app runnable), build against the facade, continue. TIER 3 — the change is IRREVERSIBLE (a one-way door: new persistent schema, destructive migration, double-charge-class money path, a competing hard-to-reverse design, or a SHARED-FOUNDATION/UPSTREAM decision: spec shape / phase boundary / shared contract is wrong) -> ESCALATE THE DECISION, not the build: write a <=1-page ADR (obstruction, 2-3 candidates with one-line tradeoffs, reversibility classification, recommendation, reversible interim), proceed on the REVERSIBLE INTERIM under a grep-able ARCH-DEBT marker, and route back FIRST-CLASS to the owning upstream step (Step 1 main-spec/phases, or Step 2 slicing) which the engine actually RE-ENTERS (re-runs the owning step, then resumes) — closing the 'escalated route-back not auto-handled in v1' gap. The reversibility classifier is the single point of failure, so the default is conservative: if the agent cannot CONFIDENTLY prove a door is two-way (reversible without data loss or consumer breakage), it is treated as Tier 3 (one-way) and escalated. Momentum never overrides this — when in doubt, escalate.
  actions: Tier 1: emit a behavior-preserving refactor commit (files-moved? assertion-changed? both checkable from the diff), verify tests green, resume. Tier 2: synthesize a new enabling slice (inserted before the current one in the phase slice list, inherits a risk tier, runs the normal pipeline), build current slice against the facade. Tier 3: write the ADR artifact to the initiative dir, drop ARCH-DEBT(slice-id): reason+ADR-link at the exact code site, build the reversible interim, and emit an UPSTREAM routeBack {command: reopen-upstream, targetStep: 1|2, reason}. The engine re-enters the named upstream step within an obstruction budget (e.g. maxUpstreamRouteBacks=2 per initiative), re-runs it, then resumes the build from the affected slice. On budget exhaustion: record a blocker, leave the reversible interim in place under its ARCH-DEBT marker, and degrade gracefully rather than loop.
  recordKeeping: Debt is recorded as CODE, twice, never only in chat: (1) a grep-able ARCH-DEBT(slice-id): reason + ADR-link marker at the exact site; (2) a committed <=1-page ADR (obstruction / 2-3 candidates+tradeoffs / reversibility / recommendation / reversible interim) in the initiative dir. Every obstruction event is appended to the run report's routeBacks/blockers accumulator with {from, tier, blastRadius, reversibility, targetStep, reason} so a hands-off run is auditable. The NEXT slice's Step 3 greps for ARCH-DEBT touching its footprint BEFORE coding, so the interim is visible to the next agent instead of being silently built upon. Recorded-in-chat or recorded-in-PR-body does not count as recorded.
--- EFFICIENCY MOVES ---
  ~ CUT the standalone research-slice step. Canonical research is folded INTO Step 3 plan as 3-6 evidence bullets, and only when surface detection fires a trigger. The unconditional research agent round-trip — the audit's flagged overhead on the majority of slices — is gone; a durable research section survives only for CRITICAL slices the verifier independently re-checks.
  ~ MERGE research+plan+build into ONE Step-3 round-trip for the fast path (TRIVIAL/STANDARD). The old pipeline spent 3 separate agent calls (research, plan, build) where a fast-path slice needs one. Build is split back out (Step 5) only for CRITICAL slices.
  ~ CUT verify-slice-plan for fast-path slices (make Step 4 conditional on CRITICAL). Plan defects on cheap slices are caught post-build by Step 6, which already re-reads plan + criteria. This removes one full agent run plus its fix loop from every cheap slice — the largest single saving across a multi-slice feature.
  ~ DRIVE depth from a single RISK field (trivial/standard/critical) computed ONCE at slice birth (Step 2), consumed everywhere downstream. A typo fix and a payments ledger no longer pay identical toll: TRIVIAL runs Step 2->3->6 (3 gates), CRITICAL runs Step 2->3->4->5->6 (5 gates) with mandatory canonical+security lenses.
  ~ COLLAPSE create-phase-spec into Step 1 (spec+phases) for small/tight phases; emit a separate phase-spec sub-pass only for L+ or risk=critical phases. A tightly-scoped single-outcome phase does not need a standalone phase-spec artifact before slicing.
  ~ MERGE verify-phase and verify-initiative for single-phase initiatives (Step 7) — run the cross-phase/readiness lenses as a delta in one pass instead of a full second verify+fix round.
  ~ FACTOR the browser/user-flow evidence standard into ONE shared 'browser evidence standard' invoked with a scope parameter (slice = this slice's UI / phase = flows spanning slices / initiative = one final screenshot), replacing the ~28 lines triplicated across three verifiers and stopping them from drifting apart.
  ~ ATTACH Stripe-grade guardrails ONLY via surface detection. A CSS or copy change triggers zero ceremony (no idempotency, no money-invariant, no migration, no boundary-parsing checks). This is the antidote to guardrail fatigue: the checklist that runs on everything gets skimmed on the one path that matters.
  ~ RECORD a single fix as a note appended to build.md rather than spawning a numbered fix-NNN.md + a fresh agent; reserve standalone fix files + dedicated agents for multi-issue or routed-back repairs.
  ~ REMOVE all stale state.mjs hints, duplicated PROJECT_LANGUAGE blocks, doubled 'ask one consequential question' lines, and redundant 'reread as if the transcript is gone' restatements — pure prose mass that is inert under the hands-off engine (no user to ask) and confuses sub-agents with a script that does not exist.
  ~ BOUND every loop AND add an initiative-level ceiling. Per-loop: maxFixAttempts, maxRouteBacks, maxSplitsPerPhase, maxUpstreamRouteBacks. Global: max total slices and max total agent calls per initiative; on approach, force consolidation or surface a blocker so a runaway decomposition degrades gracefully instead of running forever. Recovery default is revert-to-last-green carrying only a one-paragraph distilled lesson, never append correction #3 in the same context.

APPROACH: Quality-Gated-Lean: Risk-Tiered Vertical Slicing with Loop-Owned Gates
PHILOSOPHY: Keep every proven Strike gate but make each dramatically leaner and sharper, and tier the depth to the risk already computed. The 12-step linear toll (main-spec -> phases -> phase-spec -> slices -> research -> plan -> verify-plan -> build -> verify-build -> verify-phase -> verify-main-spec) is collapsed to 7 ordered steps, of which 3 are conditional and several merge research+plan or phase-spec+slices for cheap work. Rigor is never cut; verbosity is. Three things the current system buries in prose are promoted to first-class loop-owned gates: (1) ADJECTIVE-NOUN modeling (adjective is an enum/state/scope/permission/config on the existing noun until a new table is ARGUED for with distinct columns AND relationships AND independent lifecycle); (2) CANONICAL/idiomatic research, fired only by a surface-detection trigger so a CSS change pays nothing and a money/auth/dates/crypto change pays mandatory canonical lookup + package-existence check; (3) the OBSTRUCTION loop that routes a fighting-architecture moment by blast-radius and reversibility instead of hacking or spiraling. The non-negotiable backbone: the orchestration LOOP owns every stop/reassess/escalate trigger via mechanical counters; the agent never votes on whether it is stuck or done. Every gate emits a machine-readable verdict naming the failing criterion. Every artifact must pass the fresh-context bar. Verification means the one observable behavior runs through its real entry point with real data, never 'tests pass'. This directly closes the audit's single biggest gap: a first-class route-back to upstream structural steps (phase boundary, main spec) when an upstream decision is wrong, which the v1 engine literally logged as 'not auto-handled'.
--- STEP SET (7) ---
  • S0 Main Spec + Phase Map (merged, one pass) [Always.] :: Produce the durable main spec AND the smallest ordered outcome-named phase list in one artifact, so a fresh context can build without re-interviewing the user or reading the chat transcript.  [gate: PASS iff: spec has all four boundary blocks; every phase is vertical and outcome-named (no horizontal title); every new durable noun has an adjective-noun decision; estimatedSliceCount <= initiative budget (else FAIL: over-decomposition, force consolidation). FRESH-CONTEXT CHECK: re-read as if the transcript is gone -- a planner could proceed without re-interviewing.]
  • S1 Phase Spec + Slices (merged for small phases, split for large/high-risk) [Always per phase; the spec/slices MERGE for small low/standard phases and SPLIT into two passes for L or high-risk phases.] :: For one phase, define it precisely enough to slice AND emit the vertical-slice set with acceptance criteria and a per-slice risk tier -- in ONE pass for small phases, two for large.  [gate: PASS iff: every slice is Vertical or has a written Non-Vertical Justification naming the vertical slice it unblocks; first slice isTracerBullet=true; no slice has an 'and' in its one-behavior sentence; no L/XL unsplit without Why Not Split; every slice has binary acceptance criteria and a risk tier. If boundaryProblem set => ROUTE BACK to S0 (first-class upstream escape).]
  • S2 Surface-Detect + Plan (with inline evidence; canonical research folded in) [Always per slice, but research is INTERNALLY conditional on surface flags (merged into plan when no surface fires).] :: Produce a build-ready plan for one slice, with canonical/idiomatic research done ONLY when a real surface triggers it, and the adjective-noun decision recorded for any new persistent noun.  [gate: PASS iff: plan names files/surfaces/edge-cases/focused-tests; every new persistent noun has a field-not-table-defaulted decision with no un-argued new table and no UNION smell; if any money/auth/crypto/dates surface flag is true then canonicalResearch.needed=true and findings are source-backed (else FAIL); every new dependency verified to exist+popular; no new utility duplicates an existing repo home. splitNeeded => SPLIT (re-slice).]
  • S3 Verify Plan (CONDITIONAL gate -- full path only) [CONDITIONAL: runs only for M+ or money/auth/data/migration/external-API slices; SKIPPED otherwise.] :: Catch plan defects before code ONLY for slices where a post-build catch would be expensive: M+ size or any domain/security/data/migration/external-API surface.  [gate: PASS iff no Must-Fix and adjective-noun recheck ok and (if surface fired) canonical lens satisfied. FAIL with fixNeeded => fix loop (loop-owned, bounded). routeBack => upstream re-entry.]
  • S4 Build (one slice, one hat, obstruction-aware) [Always per slice.] :: Implement exactly the planned slice scope with the smallest complete path, wearing ONE hat (behavior xor refactor), and detect-then-route architectural obstruction instead of hacking or spiraling.  [gate: PASS iff built=true and hat is single (diff shows behavior xor structure, not both) and every fired surface flag has its guardrail applied. obstruction.tier set => route per the obstruction decision tree. built=false => route back to S2/S3.]
  • S5 Verify Build (honest ladder, risk-scoped lenses) [Always per slice; lens depth and rung-3 are conditional on risk tier and surface flags.] :: Confirm the ONE observable behavior actually works through its real entry point with real data, with lens depth scaled to the slice's risk tier -- never report green tests as 'verified'.  [gate: PASS iff the one behavior is exercised through its real entry point (or honestly downgraded to code-verified with residual risk listed) AND tests are non-tautological AND (UI) a screenshot was inspected AND composes with prior slices AND all fired-surface rung-3 checks pass. FAIL+fixNeeded => bounded fix loop. routeBack => upstream.]
  • S6 Verify Phase + Final (merged when single-phase) [Phase verify is always; the FINAL delta MERGES in only on the last/single phase instead of a separate 7th step.] :: Confirm slices compose into the phase outcome and, on the last phase, that phases compose into the main spec -- collapsing the two near-identical passes into one when there is a single phase.  [gate: PASS iff all required slices verified AND cross-slice composes AND (last/single phase) main-spec covered AND shipSafe. routeBack to S0/S1 is honored by the engine (re-run owning step, then resume) under the route-back budget.]
--- OBSTRUCTION LOOP ---
  detection: Loop-owned signals, NOT agent self-judgment. The harness fires a hard interrupt that injects a fixed reassess step when ANY mechanical counter trips: (a) 2 edits to the SAME target file/region with the resulting error-output hash UNCHANGED (repetition without state change); (b) 3 consecutive build turns with no acceptance criterion moving FAIL->PASS; (c) 2 failed self-corrections on one objective; (d) the build agent emits obstruction=true or proposes a change reaching OUTSIDE the slice's planned file footprint or touching a shared foundation (the refactor-spiral tripwire); (e) the diff shows both moved/renamed files AND changed assertions (two-hats violation). To blunt the gameable-fingerprint risk, the counter also tracks criterion-movement (turns since last FAIL->PASS) which a target-perturbing agent cannot dodge, and footprint-escape, which is checked from the diff not the agent's narration.
  decisionTree: On interrupt, route by BLAST-RADIUS x REVERSIBILITY (deterministic, needs no architectural taste): TIER 1 = change is in-footprint AND behavior-preserving (the slice's own files, tests green before/after) => allow a PREPARATORY REFACTOR as a separate refactor-hat commit, then resume the behavior commit. TIER 2 = needs a NEW seam but it is REVERSIBLE (two-way door) => insert an ENABLING SLICE behind a facade/interface (a new vertical slice prepended to the list), build against the seam, defer the real implementation. TIER 3 = IRREVERSIBLE (one-way door: new persistent schema, new external/cross-boundary dependency, new public contract, or competing hard-to-reverse design) OR the obstruction proves an UPSTREAM decision wrong (phase boundary / main-spec shape / shared contract) => ESCALATE THE DECISION, not the build: write a <=1-page ADR, proceed on a reversible interim under an ARCH-DEBT marker, and route back to the owning upstream step (S0/S1) which the engine RE-ENTERS. The reversibility classifier is the single point of failure, so the default is conservative: if the agent cannot prove a change is a two-way door, it is treated as Tier 3 (escalate), never silently cemented -- this resolves the 'keep moving vs stop on irreversible' contradiction in favor of safety.
  actions: Tier 1: refactor-hat commit (no behavior change, tests green both sides), then resume. Never let a Tier-1 refactor reach outside the slice footprint -- the moment it wants to, it becomes Tier 2. Tier 2: prepend one enabling vertical slice (facade + thinnest real path), build behind the seam, continue. Tier 3: write ADR {obstruction, 2-3 candidate approaches each with a one-line tradeoff, reversibility classification, recommendation, reversible interim}; drop a grep-able ARCH-DEBT(slice-id): reason + ADR-link marker at the exact site; route back to the owning upstream step under the route-back budget; the build proceeds on the reversible interim so the run is never fully blocked while the decision is async-ratified. REVERT-AND-RESET primitive (anti-sunk-cost): after 2 failed self-corrections, do NOT append a third instruction -- revert to last green and restart carrying ONLY a one-paragraph distilled lesson, or delegate to a sub-agent whose failed-attempt context is discarded on return. Accumulated failed context is itself the cause of continued failure.
  recordKeeping: Debt is recorded as CODE, twice, never only in chat (chat does not count): (1) a grep-able ARCH-DEBT(slice-id): reason + ADR-link marker at the exact site; (2) a committed <=1-page ADR. Every route-back and ADR is absorbed into the engine's structured report (routeBacks[], blockers[], assumptions[]) so a hands-off run is auditable and an async human can SEE and ratify the escalation. The NEXT slice's S2 surface-detect step greps for ARCH-DEBT touching its footprint before planning, so the next agent builds on the recorded decision, not the hack. All loops are bounded: maxRouteBacks, maxSplitsPerPhase, maxFixAttempts per loop PLUS an initiative-level ceiling (max total slices / agent calls); on exhaustion the run records a blocker and degrades gracefully (revert-to-last-green carrying the lesson) rather than looping forever.

APPROACH: Self-Correcting Strike (SC-Strike): the Obstruction Loop is the Spine
PHILOSOPHY: Center every decision on ALTITUDE + SELF-CORRECTION, but move the act of self-correction OUT of the agent's judgment and INTO the orchestration loop. The agent never votes on whether it is stuck, done, or off-track; mechanical counters owned by the engine fire hard interrupts the agent cannot rationalize away. Around that backbone hang three things, in priority order: (1) the Architectural-Obstruction Loop — the one place a build step can step back and re-route by blast-radius/reversibility instead of hacking forward or spiraling into a 40-file refactor; this is the system's spine and the current engine's single biggest correctness gap (it literally logs 'escalated route-back not auto-handled in v1'). (2) Honest verification — 'verified' means the one observable behavior ran through its real entry point with real data, never 'tests pass' or 'code looks correct.' (3) Modeling + canonical discipline — adjective-noun ('field, not table' by default) and canonical-implementation, attached only where a detection pass finds their trigger surface. The whole thing is risk-tiered: a CSS change and a payments ledger must not pay the same toll, or the agent learns to skip the checklist that guards the money path. Leanness is not a nice-to-have; ceremony-on-everything IS the failure mode the research names. So I CUT the pipeline from 12 fixed steps to a 4-to-7-step variable path: a 3-step fast lane for XS/S non-domain slices and a full lane only for M+/domain/irreversible work, with a first-class upstream route-back that the engine actually re-enters (not just logs).
--- STEP SET (7) ---
  • 0. Detect + Tier (engine-side, no agent call on the happy path) [Always (replaces standalone slice step; absorbs sizing + detection + naming-lens).] :: Compute each slice's risk tier and trigger surfaces ONCE at slice creation, so every downstream step's depth is driven by data already on disk — not recomputed per step.  [gate: Every slice has a tier and (if it names an adjective-noun) an adjNounCall. No slice titled with 'and/full/complete/MVP/setup'. No horizontal slice (a slice whose one-behavior answer is 'none yet, groundwork') — convert to a time-boxed spike whose output is a written decision. First slice is a tracer bullet piercing all layers.]
  • 1. Plan (+inline evidence, +canonical if flagged) [Always. Merges research+plan. Research is a separate prior agent call ONLY when FULL tier AND domainFlags includes external-API/payments/auth (where evidence is heavy enough to warrant isolation).] :: Produce a build-ready plan; for FAST slices this is the ONLY pre-build step (research folded in). Name 2-3 approaches with one-line tradeoffs before committing — the cheap reverse gear LLMs lack.  [gate: Plan names files+surfaces+focused tests+behavior entry point without guessing; step-back paragraph present; if domainFlag set, canonicalChecks non-empty and any new dep verified; splitNeeded handled. FAST tier: readyToBuild=true ends pre-build. FULL: readyToVerify=true -> step 2.]
  • 2. Verify Plan [SKIPPED for FAST tier (verify-build re-reads the plan and catches plan defects post-build). MANDATORY for FULL tier and any domain/security/data/migration/external-API slice.] :: Catch plan defects (wrong model, missing canonical pattern, oversized slice, duplicated utility) before any code is written — but only when the stakes justify a standalone gate.  [gate: Ready=true (all criteria PASS, mandatory lenses ran for flagged surfaces) -> build. fixNeeded -> fix loop. routeBack -> engine re-enters the named upstream step (capped by budget).]
  • 3. [gate: built=true AND oneHatOk AND (guardrails for every flagged surface present) -> verify-build. obstruction.hit -> Obstruction Loop (see loop). built=false w/ routeBack -> engine honors.]
  • 4. Verify Build (honest behavior ladder) [Always. --- OBSTRUCTION LOOP ---
  detection: The engine — not the agent's self-judgment — owns detection via TWO channels. CHANNEL A (declared, fast): build-slice emits a structured `obstruction` object whenever the planned path cannot proceed without one of four mechanical triggers: (a) editing a file OUTSIDE the slice's recorded footprint, (b) touching a shared foundation, (c) a new persistent schema/contract/public API not in the plan, or (d) a choice between competing hard-to-reverse designs. These are surface facts (file paths, schema verbs, contract edits) the engine can corroborate against the diff, NOT a feeling. CHANNEL B (mechanical stall counters, ungameable-by-construction): the loop fingerprints {tool + target-region + hash-of-resulting-error} per turn and fires a HARD INTERRUPT on: 2 same-target edits with unchanged error, OR 3 turns with no acceptance criterion moving FAIL->PASS, OR 2 failed self-corrections on one objective, OR per-loop budget exhaustion. Because a capable agent can perturb the target to dodge a single fingerprint, the counters are AND-gated with a coarse 'criteria-moved?' signal: if no criterion flips FAIL->PASS within the window, the interrupt fires regardless of how the target was perturbed — you cannot dodge 'made zero progress.' Channel A catches the architectural case the agent CAN see; Channel B catches the fixated case it can't.
  decisionTree: On obstruction, route deterministically by BLAST-RADIUS x REVERSIBILITY (no architectural taste required): TIER 1 — in-footprint AND behavior-preserving -> a PREPARATORY REFACTOR, hard-bounded to files the slice already touches, tests green before+after (two-hats: zero behavior change). Resume the build. TIER 2 — reversible NEW seam, two-way door -> insert an ENABLING SLICE behind a facade/interface; the original slice proceeds against the seam. Add the new slice to the phase list (counts against maxSplitsPerPhase). TIER 3 — irreversible (one-way door) OR competing hard-to-reverse design OR shared-foundation/upstream-decision wrong -> ESCALATE THE DECISION, not the build: write a <=1-page ADR {obstruction, 2-3 candidates w/ one-line tradeoffs, reversibility, recommendation, reversible interim}, drop a grep-able ARCH-DEBT(slice-id) marker at the exact site, PROCEED on the reversible interim, and surface the ADR for async ratification. If the wrong decision is UPSTREAM (spec shape, phase boundary, shared contract), the SAME Tier-3 path emits a first-class routeBack to create-main-spec / create-development-phases / create-phase-spec — and the engine ACTUALLY RE-ENTERS that step then resumes (fixing the current engine's 'escalated route-back not auto-handled in v1' gap). The reversibility classifier is the single point of failure, so it is conservative-by-default: ambiguous -> treat as one-way (Tier 3). Anything touching money/migrations/auth/external-effects is one-way by rule, never agent-judged.
  actions: Tier 1: behavior-preserving refactor inside footprint, then continue build. Tier 2: create facade + enabling slice, register it, build original against seam. Tier 3: write ADR + ARCH-DEBT marker + proceed on reversible interim + (if upstream) routeBack to the owning upstream step which the engine re-enters. On a Channel-B stall interrupt specifically: REVERT-AND-RESET, not continue-and-correct — discard the polluted context, restore last green state, restart the objective carrying ONLY a one-paragraph distilled lesson (or delegate to a sub-agent whose failed-attempt context is dropped on return). Never reach correction #3 in the same context window.
  recordKeeping: Debt is recorded as CODE, twice, never in chat: (1) a grep-able ARCH-DEBT(slice-id): reason + ADR-link marker at the exact site; (2) a committed <=1-page ADR artifact. The next slice's first build action greps ARCH-DEBT touching its footprint. The engine logs every obstruction to report.routeBacks/report.blockers with {from, tier, blastRadius, reversibility, command, target, reason}, and every loop is bounded: maxRouteBacks, maxSplitsPerPhase, maxFixAttempts, PLUS a new initiative-level ceiling (max total slices / max agent calls). On any budget exhaustion the run records a blocker and DEGRADES GRACEFULLY (revert to last green, surface the blocker) rather than looping forever. Resume re-reconciles in-memory slices against the on-disk slices/ dir so a mid-phase split/route-back is deterministic on replay.
--- EFFICIENCY MOVES ---
  ~ CUT the 12-step fixed pipeline to a variable 4-7 steps. FAST lane (XS/S non-domain vertical slices = the majority) = Plan -> Build -> Verify-Build (3 steps, ONE pre-build agent call). FULL lane = Plan -> Verify-Plan -> Build -> Verify-Build (+ phase/main verify). A typo fix and a payments ledger no longer pay the same toll.
  ~ MERGE research into plan. Research becomes an inline evidence note for FAST slices and a separate agent call ONLY for FULL+heavy-domain (external-API/payments/auth). Removes one full agent round-trip on most slices.
  ~ MAKE verify-plan conditional. Skip the standalone plan-verification gate for FAST slices — verify-build already re-reads the plan and catches plan defects post-build. Drops one agent run + its fix loop from every cheap slice.
  ~ COLLAPSE verify-main-spec into verify-phase when there is exactly ONE phase (run the cross-initiative readiness lenses inline as a delta). Saves an entire final verify+fix round on single-phase initiatives.
  ~ FOLD phase-spec into slicing for small, tightly-scoped phases — produce spec-and-slices in one pass; keep a standalone phase-spec only for L/risky phases.
  ~ FOLD detection+sizing+naming-lens into slice creation (computed ONCE, consumed everywhere) instead of re-deriving risk at each step.
  ~ FACTOR the browser/user-flow standard into ONE shared evidence rung invoked with a scope param (slice/phase/final) instead of three near-identical ~28-line copies that drift.
  ~ LEAN debt recording: a single in-place Must-Fix is a note appended to the existing build.md, not a fresh numbered fix-NNN.md + a new agent; reserve standalone fix files for multi-issue or routed-back repairs.
  ~ DELETE dead/stale prose: the 'ask the user' clauses (no user in a hands-off run -> 'record as assumption'), the duplicated PROJECT_LANGUAGE block, the stale state.mjs hints (the JS engine replaced state.json), and the redundant 'reread as if the transcript is gone' restatements (keep once per skill in the Quality Bar).
  ~ GATE the ceremony itself on risk: surface-detection, canonical research, and the heavy lens set run only when domainFlags fire — so the aggregate guardrail cost is near-zero on the happy path and concentrated on the money/auth/migration surfaces that actually need it.
